基础（basic）

基础（basic）

系统

isApiExist

  • 判断API是否可调用

  • 传入参数：

传入参数 说明
apiName api的名字
group group名字
  • 回调参数：
回调参数 说明
result "0"可以使用，"1"无法找到该api，"2"无权限

getSystemInfo

  • 获取系统信息

  • 无传入参数

  • 回调参数：

回调参数 说明
platform 设备平台，iOS或Android
brand 设备品牌及型号，iPhone X
version APP版本号，1.0.0
system iOS或Android系统版本，13.0.1
benchmarkLevel 设备性能等级，0、1、2、3，数值越高性能越好，性能差时可以关闭动态效果和其他费时操作
  • 说明： 以上字段，识别失败返回 nat-unknown

生命周期

  • 页面间参数传递说明

  • 假设由A页面跳转到B页面，A页面需要向B页面传递参数，则在A页面调用router-open方法中的'data'参数，在B页面的onDocumentLoaded方法的'data'接收。

  • 假设B页面完成操作后返回A页面时B页面需要给A页面传递参数，则由B页面调用navigateBack中的'data'参数，由A页面的onPageShow方法中的'data'接收。
  • 参数传递格式由A页面与B页面进行协商。

onDocumentLoaded

  • 每次webview完全加载完成时会调用，目前整个webview生命周期仅调用一次

  • 传入参数

传入参数 说明
data 前一个页面传来的参数。通过前一个页面router-open方法中的‘data’传递来的参数
  • 无回调参数

onPageShow

  • webview切回前台事件之前调用，例如从后一页面返回，APP整体从后台切回等

  • 传入参数

传入参数 说明
data 其他页面传来的参数。一般是后一个页面调用navigateBack传递来的参数，格式为字符串（可以为json字符串）
  • 无回调参数

onPageHide

  • webview切到后台事件之前调用，例如打开其他页面之前，APP直接在本页面切到后台时

  • 无传入参数

  • 无回调参数

navigateBack

  • 关闭当前webview返回上一页

  • 传入参数

传入参数 说明
message 需要借助原生APP显示的提示语。通过toast显示
data 后一个页面传递给前一个页面的信息，前一个页面将通过onPageShow收取到所传信息，格式为字符串

  • 无回调参数

  • 说明

当Web页面发现token失效时，必须调用navigateBack方法返回原生，并传递tokenInvalid参数告知原生重新登录。提示：回退到原生页面之前可以先弹框提示用户，登录状态已过期，用户已被迫下线，从而不至于使用户感到困惑。

relaunch

  • 重载当前页面

  • 传入参数

传入参数 说明
url 重载时的入口url，传空则会使用默认的入口url
  • 无回调参数
